# Sector Selector Skill

## 用途

基于知识库分析，对未来产业链板块进行**量化评分排序**，输出核心板块及其成分股列表到 JSON 文件，供量化策略调用。

> **⚠️ 硬性要求：板块代码必须100%来自akshare EastMoney API验证，严禁编造任何代码。**

## 使用方式

```markdown
/sector_selector
```

# 板块选择与股票排序分析

## 分析框架

### 一、板块四维评分模型

每个板块从 4 个维度评分（1-10 分），加权求和：

| 维度 | 权重 | 评分标准 |
|------|------|----------|
| **政策确定性** | 30% | 十五五规划明确列入 ★★★★★=10 |
| **产业链卡位** | 25% | 占据关键节点 ★★★★★=10 |
| **业绩兑现度** | 25% | 3年内可见业绩 ★★★★★=10 |
| **弹性空间** | 20% | 潜在涨幅/风险比 |

#### 评分细则

**1. 政策确定性（30%）**

| 评分 | 标准 |
|------|------|
| 10 | 十五五第一梯队 + 新质生产力核心 |
| 8-9 | 十五五第一梯队 |
| 6-7 | 十五五第二梯队 |
| 4-5 | 转型升级类 |
| <4 | 政策不明确 |

**2. 产业链卡位（25%）**

| 评分 | 标准 |
|------|------|
| 10 | 全球垄断（ASML级别）|
| 8-9 | 全球领先（宁德时代级别）|
| 6-7 | 国内领先，有全球竞争力 |
| 4-5 | 国内龙头 |
| <4 | 竞争激烈，门槛低 |

**3. 业绩兑现度（25%）**

| 评分 | 标准 |
|------|------|
| 10 | 已盈利，增速 > 50% |
| 8-9 | 已盈利，增速 30-50% |
| 6-7 | 已盈利，增速 10-30% |
| 4-5 | 亏损但收入高速增长 |
| <4 | 亏损，业绩看不清 |

**4. 弹性空间（20%）**

| 评分 | 标准 |
|------|------|
| 10 | 总市值 < 100亿，概念稀缺 |
| 8-9 | 100-300亿，概念丰富 |

### 二、股票五维评分模型

| **政策受益** | 20% | 与板块政策契合度 |

### 三、综合评分公式

```
板块综合分 = 政策确定性×0.30 + 产业链卡位×0.25 + 业绩兑现度×0.25 + 弹性空间×0.20

板块归一化分 = 板块综合分 / 所有板块最高综合分 × 100

股票综合分 = 基本面×0.30 + 政策受益×0.20 + 产业链地位×0.20 + 技术护城河×0.15 + 资金关注×0.15

盈利空间评分 = (目标价/现价 - 1) × 10  （取 1-10 之间）
风险评级 = 1-10（10=极低风险，1=极高风险）

性价比 = 盈利空间评分 × 风险评级 / 10

个股归一化分 = 股票综合分 / 板块内最高股票综合分 × 100
```

### 输出规则

1. **输出位置**：`config/sectors/sector_portfolio.json`
2. **板块数量**：知识库中实际存在的板块总数（持续增长），全部输出
3. **每板块股票数**：5-20只
4. **priority**：按综合分和爆发评分综合确定，1=最高优先级（爆发S级板块优先）
5. **version**：每次生成时更新为当天日期
6. **不需要权重**
7. **贵金属板块必须包含**，无论评分高低
8. **⚠️ 板块代码必须100%来自akshare API验证**：
   - 执行前必须调用 `ak.stock_board_industry_name_em()` 和 `ak.stock_board_concept_name_em()` 获取真实代码
   - 板块名称必须与EastMoney API返回的板块名称完全匹配
   - 不得使用任何未经验证的占位符代码
   - 如果知识库中的板块在EastMoney中找不到匹配，必须标注"未找到匹配"并跳过

### 读取示例

```python
import json

def load_sector_config(path="config/sectors/sector_portfolio.json"):
    with open(path) as f:
        config = json.load(f)

    # 获取优先板块（priority 1-2）
    preferred = [s for s in config["sectors"] if s["priority"] <= 2]

    # 获取板块内股票
    stocks_by_sector = {s["name"]: [st["code"] for st in s["stocks"]]
                         for s in preferred}

    return config, preferred, stocks_by_sector
```

## 执行流程

0. **⚠️ API代码验证（必须第一步）**：调用akshare获取真实EastMoney板块代码
   ```python
   import akshare as ak
   industry_df = ak.stock_board_industry_name_em()  # 行业板块
   concept_df = ak.stock_board_concept_name_em()    # 概念板块
   # 建立板块名称→代码映射表
   ```
1. **读取知识库**：加载产业链全景、各板块洞察、十五五政策文件、短期爆发信号文件
2. **板块匹配验证**：将知识库板块名称与EastMoney API返回的板块名称逐一匹配
   - 匹配成功：记录真实代码
   - 匹配失败：记录"未找到匹配"并跳过该板块
3. **板块初筛**：按政策确定性(≥7)和产业链卡位(≥6)过滤
4. **板块四维评分**：对候选板块打分，计算归一化分
5. **短期爆发评估**：对候选板块评近期3-6个月爆发信号，标记⭐⭐⭐爆发板块
6. **板块排序**：按归一化分降序，⭐⭐⭐板块优先
7. **股票五维评分**：对板块内所有股票打分，计算个股归一化分
8. **计算权重**：初始权重 = 个股归一化分/板块内总和；调整后权重 = 初始权重 × 板块归一化分/100
9. **计算性价比**：潜在涨幅 × 风险评级 / 10
10. **股票排序**：板块内按调整后权重降序，全市场按综合分降序
11. **生成报告**：输出 `config/sectors/sector_portfolio.json`，包含：
    - 板块评分表（EastMoney代码 + 归一化分 + 爆发等级）
    - 板块-个股完整列表（所有股票 + 权重）
    - 爆发力监测清单（⭐⭐⭐黄金组合优先）
    - 组合配置建议
